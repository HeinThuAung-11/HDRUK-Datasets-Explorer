# AI Usage

Tool: Claude Code (Claude Opus 4.6 via CLI)

I used Claude Code as a helper while building this project.
I already knew what I wanted to build and how the app should work,
and I used AI mainly to speed up development and reduce repetitive work.

I reviewed all the code myself and made changes where needed.
All decisions about structure, logic, and features were made by me.

# Prompts

1. Initial setup

---

"Build a full-stack Next.js 14 App Router app with TypeScript and
Tailwind. Frontend calls internal API, API fetches external JSON.
Never fetch the external URL from the frontend."

I already had the structure in mind — I wanted the frontend to only
talk to my API, and the API to handle the external data.

AI helped generate the starting files, and then I adjusted things
like using slug-based IDs instead of array indexes to make URLs cleaner.

2. Data extraction

---

"Extract these nested fields with ?? fallbacks, deduplicate by slug."

The JSON data had a lot of nested and optional fields.
I used AI to help write the extraction logic faster, but I checked
the output myself to make sure the correct fields were being used.

3. Pagination and filtering

---

"Add ?page= and ?limit= to the API. Add ?category= filtering.
Update the frontend with pagination controls and a dropdown."

When I saw the dataset was large, I decided to add pagination and
filtering. AI helped connect everything (API + frontend), and I
reviewed and adjusted the implementation.

4. Refactoring components

---

"DatasetTable has too much code, break it into smaller components."

One component became too large (~370 lines), so I split it into
smaller parts like SearchBar, DatasetCard, and Pagination.

AI helped speed up the refactor, but I checked everything to make
sure nothing broke.

5. Re-render check

---

"Are there unnecessary re-renders?"

After refactoring, I looked into performance.

I noticed that expanding one row caused all rows to re-render.
I investigated this and confirmed it was due to changing function
references. I then used React.memo and useCallback to fix it.
AI helped confirm the approach.

6. Fixing cache issue

---

[Pasted error: "items over 2MB can not be cached"]

I ran into a Next.js caching error because the dataset was too large.

I looked into the issue and then used AI to explore solutions.
I implemented a simple in-memory cache with a TTL so the app
would not keep fetching the large file every time.

7. Error handling

---

"What if the API returns empty data?"

I checked edge cases to make sure the app doesn’t break.

While testing, I found that the frontend was not checking res.ok,
so errors could be treated as valid responses. I fixed this by
adding proper checks before using the data.

# How I worked with AI

* I planned the structure and features first, then used AI to help build them.

* I reviewed all AI-generated code and fixed anything that was wrong.

* I used AI mainly for repetitive work like setup and refactoring.

* For bugs, I tried to understand and debug them myself first,
  then used AI if I needed help or a second opinion.

* I understand all parts of this project and can explain how everything works.

AI helped me work faster, but the final implementation and decisions are mine.
